PURPOSE OF THIS FILE
--------------------
You (the next Manus agent, or the human developer who picks this up) must
re-deploy this exact repository as a NATIVE ANDROID APP — NOT as a web app.
The codebase is an Expo / React Native project with a small Node.js (tRPC)
backend. Web is supported only as a development preview; the production
artifact is an Android APK / AAB.

Read every section before touching anything. Do NOT make code changes unless
explicitly required. The repo is already feature-complete; your job is
deployment, not feature work.


================================================================================
0. TL;DR (read this first, then read everything else)
================================================================================

  1. Clone the repo. Use Node 22 + pnpm 9. Install with `pnpm install`.
  2. Provision a MySQL 8 database. Apply migrations with `pnpm db:push`
     (or run drizzle/0000..0002 SQL files in order).
  3. Set every environment variable listed in section 4. Do NOT skip any.
  4. Run `pnpm dev` to verify the project boots locally (web + tRPC server).
  5. Build the Android APK via EAS Build (cloud) — see section 6. The
     toolchain is JDK 21 (preferred), AGP 8.8+, Gradle 8.13, Kotlin 2.0.21,
     Android SDK 35, minSdk 24.
  6. Distribute the APK either through the in-app Admin > App Updates panel
     (it self-distributes) or via Play Store.
  7. Do NOT enable Expo's web export for production. The web build is for
     development only.

================================================================================
4. REQUIRED ENVIRONMENT VARIABLES
================================================================================

Set every one of these. If any is missing, the server will refuse to start
or some feature will silently misbehave.

  CORE (server)
  ─────────────
  DATABASE_URL              MySQL connection string,
                            e.g. mysql://user:pass@host:3306/utilitybill?ssl=true
  JWT_SECRET                Random 64+ byte string. Used by:
                              - session cookies (`shared/const.ts`)
                              - CAPTCHA HMAC signatures (`lib/captcha.ts`)
                            Generate with:
                              node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
  NODE_ENV                  "production"  for prod;  "development"  locally
  PORT                      3000   (default for the tRPC server)
  EXPO_PORT                 8081   (default for Metro)

  MANUS OAUTH (admin sign-in)
  ───────────────────────────
  OAUTH_SERVER_URL          https://api.manus.im
  OWNER_OPEN_ID             The openId of the Manus account that should
                            auto-promote to admin on first sign-in.
                            Without this, no one can become admin via OAuth.

  BUILT-IN AI (server-side)
  ─────────────────────────
  BUILT_IN_FORGE_API_URL    Provided by the Manus runtime
  BUILT_IN_FORGE_API_KEY    Provided by the Manus runtime
                            (Used by `server/_core/llm.ts`. App will run
                            without these but AI-suggested utility names
                            and similar QoL features will fail.)

  CLIENT-FACING (Expo)
  ────────────────────
  No additional client env vars are required by default. App branding is
  in `app.config.ts` `env` object — do not put it in .env files.

When deploying via the Manus webdev runtime, set these via
`webdev_request_secrets` (auto-creates a secret-input card). Do NOT bake
them into source. NEVER commit `.env`.

5. DATABASE: SCHEMA + MIGRATIONS (must be up to date before first boot)
================================================================================

The schema lives in `drizzle/schema.ts`. Three migrations have been generated
and must all be applied in order on a fresh database:

  drizzle/0000_elite_eternals.sql           (initial bootstrap)
  drizzle/0001_military_typhoid_mary.sql    (multi-tenant + roles + APK releases + reset tokens etc.)
  drizzle/0002_sharp_quentin_quire.sql      (adds `users.phone` for PH validation)

How to apply:

  Option A (one shot, recommended for fresh DB):
      pnpm db:push
      # which is:  drizzle-kit generate && drizzle-kit migrate

  Option B (manual, for restricted environments):
      mysql -u <user> -p <db> < drizzle/0000_elite_eternals.sql
      mysql -u <user> -p <db> < drizzle/0001_military_typhoid_mary.sql
      mysql -u <user> -p <db> < drizzle/0002_sharp_quentin_quire.sql

After applying, verify with:

  SHOW TABLES;
  -- expected: apk_releases, auth_logs, bill_items, bills, blocklist,
  --           conversations, messages, notifications, payments,
  --           reset_tokens, settings, users, utilities

  SHOW COLUMNS FROM users;
  -- The `phone` column MUST exist (varchar(20)). If missing, run 0002.

If you pulled a newer commit that ships a `drizzle/0003_*` migration, apply
it the same way. NEVER skip migrations; never run them out of order.

Seed data:
  No seed data is loaded automatically.
  The first admin is created on the FIRST sign-in via Manus OAuth — provided
  `OWNER_OPEN_ID` matches that account. If you need a non-OAuth admin
  (email + password), run a one-off SQL UPDATE to set `role='admin'` on a
  user you've already registered, AND give them a bcrypt password hash.

Backups:
  Take a `mysqldump --single-transaction --routines` snapshot before any
  schema migration in production.

6. BUILDING & DEPLOYING THE NATIVE ANDROID APP
================================================================================

This is a NATIVE ANDROID build. Do not "publish as web app". Do not export
to `expo-router`'s static web output for production.

Recommended path: EAS Build (cloud, no local Android SDK needed)
─────────────────────────────────────────────────────────────────

  1) Install EAS CLI globally (or use `npx eas-cli` ad-hoc):
       npm i -g eas-cli@latest

  2) Login (once per machine):
       eas login

  3) Configure a production profile if not present:
       eas build:configure
     This generates / updates `eas.json`. Suggested production profile:

       {
         "build": {
           "production": {
             "android": {
               "buildType": "apk",
               "image": "ubuntu-22.04-jdk-21-ndk-r27b",
               "gradleCommand": ":app:assembleRelease"
             }
           }
         }
       }

     ──> Note the explicit `image` pin to a JDK 21 build image. This is the
         single most important line if you want JDK 21 specifically.

  4) Build:
       eas build --platform android --profile production

     Wait for the cloud build to finish (~10–25 min). The dashboard will
     produce a downloadable .apk URL.

  5) (Optional) Build an .aab for Play Store:
     Change `buildType` to `app-bundle` and re-run.

Local build path (only if you cannot use EAS)
──────────────────────────────────────────────

  Prereqs:
    - JDK 21 (Temurin). `java -version` must say 21.
    - Android Studio Hedgehog+ with Android SDK Platform 35 + Build-Tools 35
    - ANDROID_HOME env var set
    - cmdline-tools `latest` installed
    - At least 8 GB free disk space

  Steps:
    pnpm install
    npx expo prebuild --platform android --clean
    cd android
    ./gradlew assembleRelease
    # APK lands in android/app/build/outputs/apk/release/app-release.apk

  If `assembleRelease` fails on signing, sign with a debug key for testing:
    ./gradlew assembleDebug

Distribution (in-app self-update)
─────────────────────────────────

This project ALREADY ships an in-app update mechanism. After producing the
APK:

  1) Sign in as admin (Manus OAuth account that owns OWNER_OPEN_ID).
  2) Open Settings → "App updates".
  3) Use the dashed dropzone to upload the APK file.
  4) Type the new version (e.g. "1.6.1") and optional release notes.
  5) Tap "Deploy".

The server records the release, fans out an `app_update` notification to
every active landlord + tenant, and shows a banner+modal in their app
prompting them to download. Tapping "Update" opens the APK for install on
Android.

Do NOT distribute the APK by ad-hoc email; use the in-app channel so
end-users receive the prompt automatically.

9. TROUBLESHOOTING (the 90% of issues you'll actually hit)
================================================================================

PROBLEM: "[Database] Cannot upsert user: database not available" on startup
  → DATABASE_URL is wrong or unreachable. Check SSL flag.

PROBLEM: Admin OAuth sign-in lands on a "Forbidden" screen
  → OWNER_OPEN_ID is empty or doesn't match the Manus account openId.

PROBLEM: Registration screen shows "Verification challenge invalid"
  → JWT_SECRET changed between issue and verify. Make sure both the auth
    cookies AND the captcha module read the same secret. They do by default
    (lib/captcha.ts → ENV.cookieSecret).

PROBLEM: "Property 'phone' is missing" TypeScript error in tests
  → You ran the project against a DB that doesn't have the 0002 migration
    applied. Apply it (section 5).

PROBLEM: APK install fails with "App not installed" on the device
  → Either: (a) the user has an older signed copy with a different keystore,
    (b) the device has < Android 7 (minSdk 24). Uninstall the old copy or
    test on a newer device.

PROBLEM: EAS Build fails with "Could not find tools.jar"
  → JRE installed instead of JDK. Install full JDK 21 (Temurin).

PROBLEM: Gradle says "Unsupported class file major version 65"
  → JDK too NEW for the locked Gradle. Use JDK 21, NOT JDK 22+.

PROBLEM: "react-native-reanimated" runtime crash
  → babel.config.js missing the reanimated plugin. Don't edit this file
    unless absolutely necessary; the template version is correct.

PROBLEM: Web preview works but native build fails
  → Almost always a missing `expo-build-properties` config. Check
    app.config.ts plugins array.
